fix(logging): Oprava formátování errorů a shutdown logování

Řeší problémy s neformátovanými error výpisy a neukládáním shutdown logů do databáze.

- **fix(log): Oprava formátování error zpráv s prefix**
  - Log.error() nyní zobrazuje formátované zprávy místo raw stack trace
  - Stack trace se zobrazuje pouze v debug módu
  - Konzistentní formátování se všemi ostatními log metodami
- **fix(shutdown): Oprava logování shutdown eventu do log_system**
  - Přesunuto logování PŘED zavření databázového spojení
  - Eliminuje problém s neukládáním shutdown záznamů
  - Přidáno debug logování pro ověření úspěšnosti
- **feat(debugger): Vylepšené přerušování aktivních session**
  - Kritické errory nyní správně přeruší aktivní waitForUserInput
  - Přidán tracking currentInputResolver pro čisté přerušení
