fix(db): Restore flexible database connection logic

- Re-implemented the database connection logic in `iv_sql.js` to correctly handle both development and production environments.
- The system now prioritizes environment variables (`CLAUDE_DB_USER`, `CLAUDE_DB_PASS`) for the connection.
- If environment variables are not found, it falls back to using the `sql_config.json` file, which is necessary for production VMs.
- This fixes a critical bug that caused the application to crash in environments without the database environment variables set.
