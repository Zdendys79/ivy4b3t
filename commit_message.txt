fix(worker): Sjednocení FB error handling a eliminace "šílených dualit"

Řeší nekonzistentní chování při různých typech FB chyb.

- **fix(browser-closure): Sjednocení uzavírání prohlížeče**
  - FB initialization failed nyní také zavírá celý prohlížeč
  - Všechny FB chyby (login i initialization) mají konzistentní chování
  - Eliminuje 10minutové čekání u initialization errors

- **fix(account-blocking): Rozšíření account blocking logiky**
  - Account blocking nyní zahrnuje FB initialization failed chyby
  - Odstranění duplikované logiky mezi tick() a executeUserActionCycle
  - Nový lock reason: "Selhání FB inicializace" s typem FB_FAILURE

- **fix(error-details): Podrobnější error zprávy**
  - FB initialization errors nyní obsahují detaily z page analysis
  - Místo "final status: warning" zobrazuje konkrétní problemy
  - Lepší diagnostika pro debugging FB problémů
